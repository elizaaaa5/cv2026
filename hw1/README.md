# Домашнее задание 1: Стабилизация видео

## Вариант: A (Мини-система стабилизации камеры)

---

## Описание решения

Данная программа реализует базовую систему стабилизации видео с использованием оптического потока Lucas-Kanade и аффинных трансформаций.

### Основные этапы:

1. **Чтение видео** - загрузка кадров из входного видеофайла
2. **Вычисление оптического потока** - используется метод Lucas-Kanade для оценки движения между кадрами
3. **Оценка глобального движения** - построение траектории движения камеры
4. **Сглаживание траектории** - применение скользящего среднего для удаления высокочастотной дрожи
5. **Компенсация движения** - применение аффинных трансформаций к каждому кадру
6. **Сохранение результата** - запись стабилизированного видео

---

## Параметры

| Параметр | Значение | Описание |
|----------|----------|----------|
| `WINDOW_SIZE` | 30 | Размер окна для скользящего среднего |
| `winSize` | (21, 21) | Размер окна для Lucas-Kanade |
| `maxLevel` | 3 | Число уровней пирамиды |
| `criteria` | (30, 0.01) | Критерий остановки итераций |

---

## Применённые фильтры

### 1. Оптический поток Lucas-Kanade
- Разреженный метод оптического потока
- Работает на основе предположения о постоянстве скорости в окне
- Хорошо подходит для видео с текстурными объектами

### 2. Скользящее среднее (Moving Average)
- Простой низкочастотный фильтр
- Убирает высокочастотную дрожь камеры
- Параметр `WINDOW_SIZE` определяет степень сглаживания

### 3. Аффинная трансформация
- Компенсирует сдвиг и небольшие повороты
- Быстрее чем перспективная трансформация

## Основные наблюдения и анализ ошибок

### Что хорошо стабилизируется:
- Случайная высокочастотная дрожь камеры
- Плавные движения при правильном размере окна
- Видео с хорошей текстурой

### Проблемы и ограничения:

1. **Отсутствие текстуры** - метод LK требует текстурных областей для корректной работы
2. **Большие движения** - при быстром движении камеры могут возникать артефакты
3. **Motion blur** - размытие движения затрудняет вычисление оптического потока
4. **Глобальное движение vs локальное** - аффинная модель не учитывает перспективные искажения
5. **Насыщение/тени** - области с одним цветом (небо, стены) не дают информации о потоке

### Пути улучшения:

- Использовать фильтр Калмана вместо скользящего среднего
- Применять перспективную модель (гомографию) вместо аффинной
- Использовать Farnebäck для плотного потока
- Добавить детектор качества кадра и пропускать размытые кадры

## Выводы

Разработанная система стабилизации демонстрирует базовый подход к решению проблемы дрожи камеры. Метод Lucas-Kanade в сочетании со скользящим средним позволяет убрать высокочастотные колебания, сохраняя при этом плавное целенаправленное движение камеры. Для более качественной стабилизации в реальных приложениях рекомендуется использовать комбинированные методы с детекцией особенностей (feature tracking) и более сложными моделями трансформаций.
